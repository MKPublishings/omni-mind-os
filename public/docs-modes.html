Cognitive Modes

Specialized reasoning frameworks and automatic mode selection

What Are Modes?

Cognitive modes are specialized mental frameworks that shape how Omni Mind/OS processes information and generates responses. Each mode embeds distinct reasoning strategies, output formats, and heuristics optimized for specific problem domains.

Mode Philosophy

Rather than training separate models, we use cognitive priming‚Äîinjecting mode-specific system prompts that steer the underlying AI's attention and reasoning patterns. This approach allows dynamic switching without model retraining.

Automatic Mode Selection

When mode is set to "Auto," Omni Mind/OS analyzes incoming messages using keyword heuristics and intent classification to select the optimal cognitive framework.

Selection Algorithm

function selectMode(message: string): CognitiveMode {
  const keywords = {
    architect: ['design', 'architecture', 'system', 'plan', 'structure', 'organize'],
    analyst: ['analyze', 'compare', 'data', 'performance', 'metrics', 'evaluate'],
    visual: ['design', 'ui', 'ux', 'layout', 'color', 'style', 'diagram'],
    lore: ['explain', 'history', 'context', 'why', 'background', 'story']
  };

  const scores = Object.entries(keywords).map(([mode, words]) => ({
    mode,
    score: words.filter(w => message.toLowerCase().includes(w)).length
  }));

  const winner = scores.reduce((a, b) => a.score > b.score ? a : b);

  // Default to Analyst if no clear signal
  return winner.score > 0 ? winner.mode : 'analyst';
}

Mode Priming System

Each mode injects a specialized system prompt that establishes the cognitive framework before processing user input.

Architect Primer

You are operating in ARCHITECT mode‚Äîa systems thinking specialist.

Focus on:
- Hierarchical decomposition of complex problems
- Identifying dependencies and relationships
- Trade-off analysis between design alternatives
- Long-term maintainability and scalability

Output format:
- Start with high-level overview
- Use visual diagrams (ASCII, Mermaid) when helpful
- Structure responses as nested hierarchies
- Highlight critical decision points

Analyst Primer

You are operating in ANALYST mode‚Äîa data-driven reasoning specialist.

Focus on:
- Quantitative metrics and performance characteristics
- Evidence-based comparisons
- Statistical patterns and trends
- Objective evaluation criteria

Output format:
- Lead with key findings and metrics
- Use tables for comparative analysis
- Cite sources and confidence levels
- Provide concrete numbers when available

Output format:

- Use ASCII diagrams for mockups

- Provide CSS/styling recommendations

- Begin with contextual background

- Synthesize disparate knowledge sources

Manual Mode Selection

Users can override automatic mode selection via the settings interface or per-message mode hints.

Via Settings

Navigate to Settings ‚Üí Model & Mode ‚Üí Cognitive Mode ‚Üí Select preferred mode

Per-Message Hints

// Prefix message with mode hint
"[architect] Design a caching layer for this API"

// System detects hint and forces Architect mode

API Parameter

POST /api/omni/stream
{
  "message": "Explain React hooks",
  "mode": "lore",  // Force Lore mode
  "model": "omni"
}
